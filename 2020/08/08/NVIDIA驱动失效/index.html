ubuntu NVIDIA 驱动失效

Posted on 2020-08-08 | Edited on 2020-08-15

背景

  • 因为某些原因我们常常需要更新ubuntu的内核，或者说它自己更新的时候更新了内核，这时候，我们之前装的驱动就会失效。需要我们的重新配置驱动。
  • 即： NVIDIA-SMI has failed because it couldn’t communicate with the NVIDIA driver. Make sure that the latest NVIDIA driver is installed and running.

内核更新的原因

  • 系统自动更新
  • 手动 apt-get 更新
  • 不同的内核更新的过程处理是不一样的！！！， 不同的内核更新的过程处理是不一样的！！！， 不同的内核更新的过程处理是不一样的！！！。重要的事情说三遍。网上很多的教程针对的都是系统更新内核的处理方式，比如卸载重装，或者至多加上 dkms进行一定的修复，但是这些操作对于手动更新的内核都是无效的。
  • 因为我们手动更新内核的时候，下载下来的是直接编译好的内核代码，而不是内核源码。但是所有外部驱动的编译都需要和内核源码一起编译的，所以如果没有内核源码，那么那些方法都会失效，即使你apt-get 成功安装了驱动也没有用，它是不会挂载到内核上去的。问题不会得到解决

  • 如果没有内核源码，采用dkms编译的时候，就会出现如下错误

    Bad return status for module build on kernel: 5.4.0-42-generic on Ubuntu18.04. NVIDIA-SMI failed (nvidia-driver-435)

解决

  • 这个解决方案，对于手动编更新和自动更新内核这两个起因都适用，甚至都不需要你卸载你原先的驱动。同样熟悉了这个方法之后，你甚至可以为你的ubuntu 内核增加任意的外部驱动。

  • dkms status 　　检查当下我们手动编译过的外部驱动，如果之前没有任何话手动编译外部驱动的经历，应该就是没有返回

  • uname -a　　 查看内核版本

    Linux shicaiwei 5.4.0-42-generic #46~18.04.1-Ubuntu SMP Fri Jul 10 07:21:24 UTC 2020 x86_64 x86_64 x86_64 GNU/Linux

  • cd /usr/src　　 查看你目前有没有这个内核的源码，内核的头文件，和你要安装的显卡驱动的源码

    • 内核源码，一般是linux-xxx,如：linux-5.4
    • 内核源码的头文件 ，一般是linux-header-xxx,如： linux-headers-5.4.0-42-generic
    • 驱动源码，一般是 nvidia-xxx,如：nvidia-440.100

  • 驱动源码

    • 一般而言,如果之前装过驱动,那么是会有驱动的源码的,如果没有,那么就需要重新装一遍驱动,直接按照这个链接 的方法1进行安装就好

  • 内核头文件安装

    sudo apt-get install --reinstall   linux-headers-$(uname -r)

  • 内核源码下载

    • 到这个地址下载你当下内核对应的源码
    • 下载的时候，只要保证大版本号，次版本号对应就好，比如我的内核版本是 ： 5.4.0-42-generic 。下载的内核源码是 5.4.0 的内核源码就行。如果这个列表没有，那么可以直接浏览器搜索，一般很容易搜到。

  • 万事具备，这时候，按照dkms的方法编译就可以成功了。

    • sudo dkms install -m nvidia -v 410.78 #410.78是安装驱动的版本

  • dkms status 检查安装结果

    • dkms 有三个状态 add、build、install。install才表示已经挂载到了内核当中。

    • 如 nvidia, 440.100, 5.4.0-42-generic, x86_64: installed

  • reboot 重启，就可以看到已经工作了的nvidia-smi

后续

  • 按照这个逻辑，用户完全可以把任何想要编译的外部驱动 源码放置在 /usr/src下，然后按照这个逻辑进行编译，安装
